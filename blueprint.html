Access Control

Role-Based Hierarchy

Admin
System Administrator

Full organization-wide access. Manages the entire workforce structure and has visibility into all data.

  • ✓Create & manage departments
  • ✓Create employees & assign roles
  • ✓Assign employees to managers
  • ✓View all attendance records
  • ✓Approve / reject any leave
  • ✓Generate organization-wide reports
  • ✓Manage leave balances
Manager
Team Manager

Scoped to their assigned team. Cannot see data outside of employees reporting to them.

  • ✓View team member profiles
  • ✓View team attendance records
  • ✓Approve / reject team leave requests
  • ✓View team leave history & balances
  • ✓View team dashboard & daily status
  • ✓Mark own attendance
  • ✓Apply for own leave
Employee
Standard Employee

Access restricted to personal data only. Cannot view colleagues' records or organizational data.

  • ✓View own profile
  • ✓Mark daily attendance (check-in/out)
  • ✓View own attendance history
  • ✓Apply for leave
  • ✓View own leave history & balance

Defense in Depth

Security Architecture

JWT + Refresh Token Rotation
Access tokens expire in 15 minutes. Refresh tokens (7 days) live in an httpOnly; Secure; SameSite=Strict cookie — never in localStorage. On each refresh, the old token is revoked and a new one is issued.
Password Hashing
bcryptjs with salt rounds of 12. Passwords are stripped from all API responses using Mongoose select: false. Password change requires providing the current password.
Account Lockout
After 5 consecutive failed logins, the account is locked for 30 minutes. failedLogins counter resets on successful login. Admins can unlock accounts manually.
Rate Limiting
Login endpoint: 5 requests / 15 min per IP. General API: 100 requests / 15 min per authenticated user. Headers communicate remaining quota. Uses express-rate-limit.
HTTP Security Headers
Helmet.js sets Content-Security-Policy, X-Frame-Options: DENY, X-Content-Type-Options: nosniff, Strict-Transport-Security, and removes X-Powered-By.
NoSQL Injection Prevention
express-mongo-sanitize strips $ and . from user-supplied input before it reaches Mongoose. Paired with Mongoose strict mode — undefined schema paths are discarded.
RBAC Middleware
Every protected route runs two middleware in sequence: authenticate (verifies JWT) then authorize(...roles) (checks role enum). Managers also get a scope check — requests for other teams' data are rejected with 403.
Input Validation
Every POST/PATCH body is validated against a Joi schema before reaching the controller. Validation failures return a structured 400 with field-level error messages. No unvalidated data reaches the business layer.
CORS Configuration
CORS is configured with an explicit whitelist of allowed origins (no wildcard in production). Credentials mode is enabled only for the frontend origin. Preflight responses are cached.
Audit Logging
All Create, Update, Delete operations are logged with Winston: actor userId, role, endpoint, payload shape (not values), and timestamp. Logs persist to rotating files for compliance review.

Engineering Decisions

Key Technical Decisions

Live Timer — Accurate Across Page Refreshes
The timer is not stored in frontend state alone. On check-in, the server saves checkIn.time (ISO timestamp) to the database. On every page load, the employee dashboard calls GET /attendance/today. If a check-in exists with no check-out, the frontend computes elapsed = Date.now() - new Date(checkIn.time) and starts the timer from that point. The displayed seconds update via a setInterval in a custom useTimer hook. No server polling needed — the truth lives in the DB.
Geolocation — Browser API + Server Validation
The useGeolocation hook calls navigator.geolocation.getCurrentPosition() and wraps it in a Promise. Coordinates are sent with the check-in/check-out request body. The server stores raw {lat, lng} and optionally reverse-geocodes to a human-readable address using an offline library or a free geocoding API. Leaflet.js displays a pin on the attendance detail page.
Manager Scope Enforcement — Backend, Not Frontend
The frontend simply does not render out-of-scope links. But the real enforcement is on the server: every manager request to /attendance/team or /leaves/team is filtered by { managerId: req.user._id } in the Mongoose query. Even if a manager crafts a raw API call to another employee's ID, the RBAC scope middleware checks that the requested employee's managerId matches the current user — otherwise returns 403.
Axios Token Refresh Interceptor
An Axios response interceptor catches 401 errors. It calls POST /auth/refresh (using the httpOnly cookie automatically), stores the new access token in memory (not localStorage), and retries the original request. This means the user is never prompted to log in again during a session — token refresh is invisible. Access tokens are stored in a module-level variable, not React state, to survive re-renders.
Leave Approval Side Effect
When a manager approves a leave, the server runs a transaction: (1) update Leave.status = 'approved', (2) create or update an Attendance record for each day in the range with status: 'leave', (3) deduct from LeaveBalance. All three writes happen atomically using MongoDB sessions. If any step fails, all changes are rolled back. This keeps attendance and leave data consistent.
Real-time Notifications via Socket.io
Each authenticated socket connection joins a private room named by userId. When a leave is approved/rejected, the server emits a leave:updated event to the employee's room. The frontend useSocket hook subscribes on mount and triggers a React Query cache invalidation, so the UI updates without a manual refresh. Admin also gets live counters (present/absent) via a broadcast on check-in events.
Standard API Response Shape
Every endpoint returns a consistent envelope via the apiResponse utility: {"success": true, "data": {...}, "message": "..."} on success, and {"success": false, "error": "...", "details": [...]} on failure. The global error handler in errorHandler.js converts all thrown AppError instances and Mongoose validation errors to this shape. Frontend Axios interceptors read response.data.data uniformly.
